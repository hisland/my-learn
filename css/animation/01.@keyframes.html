Syntax:

@keyframes  {
  [ [ from | to |  ] [, from | to |  ]* block ]*
}

animation, from和to至少有1个,否则invalid

from-to, 从from到to, 然后恢复当前状态

只有from, 从from到当前状态

只有to, 从当前状态到to, 再恢复当前状态

只有最后一个animation设置起效,不会层叠合并

关键帧值有缺失的时候采用插值, 根据time-function插入

key的语法可以是 key,key{rules} 用,号分隔不同的时间段

mdn说只有from,to里有的属性才动画,实测只要在frame有的都会动画,初始和结束值是当前状态值

关键帧的定义不需要按from-to的顺序,可以先定义80%,再定义20%

相同帧定义会采取最后一个,且相同帧的不同属性不会合并, 合并不合并的标准正在讨论中...
